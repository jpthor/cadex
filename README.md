# Cadex

Generic parametric CAD designer. Tauri 2 desktop shell, React + Three.js viewport, Rust backend, optional OpenAI tool calling, and CAD export paths.

See `PLAN.md` for the full v1 brief.

## Stack

- **Shell:** Tauri 2 (`src-tauri/`)
- **Frontend:** React 19 + TypeScript + Vite (`src/`)
- **Viewport:** Three.js (`src/geometry.ts`, `src/App.tsx`)
- **Geometry kernel (export path):** OpenVSP (script generated by Rust, run if `vsp`/`openvsp` is on `PATH`)
- **AI copilot (optional):** OpenAI Responses API with function tool calling

## Prerequisites

- Node.js 18+
- Rust toolchain (`rustup` stable)
- macOS, Windows, or Linux Tauri 2 prerequisites - see <https://tauri.app/start/prerequisites/>
- Optional: [OpenVSP](https://openvsp.org) on `PATH` for STEP export

## Install

```bash
npm install
```

## Run

```bash
npm run tauri:dev
```

This starts Vite on `http://127.0.0.1:1420` and launches the Tauri window.

## Build

```bash
npm run build         # type-check + frontend bundle into dist/
npm run tauri:build   # full desktop bundle in src-tauri/target/release/
```

## Test

```bash
cd src-tauri && cargo test    # Rust unit tests for prompt parsing, STL, OpenVSP script
```

## Using the app

1. Launch the app (`npm run tauri:dev`).
2. Type a prompt like `create a 40mm diameter round solid, 120mm long, on the XZ plane` and press **Build**.
3. The part appears in the canvas, the dependency tree updates, and the timeline records the action.
4. Click **STL** to export an ASCII STL into the app data directory's `exports/` folder.
5. Click **STEP** to either run OpenVSP (if installed) and export STEP, or save a `.vspscript` you can run later.

### Optional: OpenAI copilot

Paste an OpenAI API key into Settings and press **Build**. Cadex calls the Responses API with these tools:

- `create_part` - generic parametric CAD part from an operation, profile sketch, path, and dimensions
- `create_reference_geometry` - construction planes, points, lines, faces, and surfaces
- `orient_part` - place existing bodies on origin planes or anchors
- `inspect_geometry` - return a description of the current geometry

The API key is stored in browser local storage for the current build.
